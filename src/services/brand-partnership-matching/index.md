# Brand Partnership Matching Service

## Purpose
The Brand Partnership Matching Service is a microservice designed to match creators with relevant brand sponsorship opportunities. It utilizes machine learning-based scoring algorithms to analyze audience demographics, content themes, engagement metrics, and brand criteria for effective matching.

## Usage
To use the service, a client sends match requests that include creator IDs, optional brand IDs, and various filters. The service processes these requests to return a list of suitable brand partnerships based on specified criteria.

## Parameters/Props

### Match Request
- **creatorId**: `string` (UUID) - The unique identifier of the creator requesting brand matches.
- **brandIds**: `array` of `string` (UUID) - Optional. Specific brand IDs to match against.
- **filters**: `object` - Optional. Contains criteria for filtering matches:
  - **minAudienceSize**: `number` - Minimum audience size (optional).
  - **maxAudienceSize**: `number` - Maximum audience size (optional).
  - **targetDemographics**: `array` of `string` - Specific target demographics (optional).
  - **contentCategories**: `array` of `string` - Content categories to filter by (optional).
  - **budgetRange**: `object` - Budget constraint with:
    - **min**: `number` - Minimum budget (required).
    - **max**: `number` - Maximum budget (required).
  - **engagementThreshold**: `number` - Minimum engagement percentage (0-100) (optional).
- **limit**: `number` - The maximum number of matches to return (default: 10, min: 1, max: 100).

### Bulk Match Request
- **creatorIds**: `array` of `string` (UUID) - List of creator IDs (max: 50).
- **filters**: `object` - Same as Match Request filters.
- **limit**: `number` - Maximum number of matches to return per creator (default: 5, min: 1, max: 50).

## Return Values
The matched results will include an array of potential brand sponsorship opportunities that align with the provided criteria. Each result will likely contain details on the brand and the match score generated by the scoring algorithms.

## Examples

### Single Match Request
```typescript
const matchRequest = {
  creatorId: "123e4567-e89b-12d3-a456-426614174000",
  filters: {
    minAudienceSize: 1000,
    targetDemographics: ["18-24", "25-34"],
    budgetRange: {
      min: 500,
      max: 5000
    }
  },
  limit: 10
};
// Call to the service to get matches
const results = await brandPartnershipService.match(matchRequest);
```

### Bulk Match Request
```typescript
const bulkMatchRequest = {
  creatorIds: ["123e4567-e89b-12d3-a456-426614174000", "123e4567-e89b-12d3-a456-426614174001"],
  filters: {
    minAudienceSize: 500,
    engagementThreshold: 50
  },
  limit: 5
};
// Call to the service to get bulk matches
const bulkResults = await brandPartnershipService.bulkMatch(bulkMatchRequest);
```

The service is designed for scalability and can handle multiple requests efficiently, ensuring that creators find the best match for brand collaborations.
