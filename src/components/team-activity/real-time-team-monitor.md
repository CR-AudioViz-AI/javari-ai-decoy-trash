# Real-Time Team Activity Monitor

## Purpose
The Real-Time Team Activity Monitor is a React component designed to visualize and manage team activities in real-time. It displays agents' statuses, tasks, and communications, allowing for efficient team management and monitoring.

## Usage
To use the Real-Time Team Activity Monitor, import the component into your React application and include it in your JSX. The component is designed as a client-side component and leverages various UI components and animations.

```tsx
import RealTimeTeamMonitor from '@/components/team-activity/real-time-team-monitor';

// Inside your component
<RealTimeTeamMonitor />
```

## Parameters / Props
The `RealTimeTeamMonitor` component does not accept any props directly but manages its internal state and side effects using React hooks. It handles various activities generated by agents and tasks.

### Agent Interface
- **id**: `string` - Unique identifier for the agent.
- **name**: `string` - Name of the agent.
- **avatar**: `string` (optional) - URL for the agent's avatar.
- **status**: `'active' | 'idle' | 'offline'` - Current status of the agent.
- **currentTask**: `string` (optional) - Task currently assigned to the agent.
- **completedTasks**: `number` - Count of tasks completed by the agent.
- **efficiency**: `number` - Efficiency rating of the agent (0-100).
- **lastActivity**: `Date` - Timestamp of the last activity by the agent.

### Task Interface
- **id**: `string` - Unique identifier for the task.
- **title**: `string` - Title of the task.
- **description**: `string` - Detailed description of the task.
- **assignedTo**: `string[]` - List of agents assigned to the task.
- **status**: `'pending' | 'in_progress' | 'completed' | 'blocked'` - Current status of the task.
- **priority**: `'low' | 'medium' | 'high' | 'critical'` - Priority level of the task.
- **progress**: `number` - Progress percentage of the task (0-100).
- **deadline**: `Date` - Deadline for task completion.
- **createdAt**: `Date` - Timestamp for when the task was created.

### Communication Interface
- **id**: `string` - Unique identifier for the communication.
- **fromAgent**: `string` - ID of the agent who sent the message.
- **toAgent**: `string` (optional) - ID of the agent who received the message.
- **type**: `'direct' | 'broadcast' | 'task_update' | 'system'` - Type of communication.
- **message**: `string` - Content of the message.
- **timestamp**: `Date` - When the communication took place.
- **taskId**: `string` (optional) - Identifier of the related task.

## Return Values
The component returns a JSX element that visualizes team activities including agent statuses, task progress, and communication logs. It does not return any data directly as it is self-contained.

## Examples
```tsx
import React from 'react';
import RealTimeTeamMonitor from '@/components/team-activity/real-time-team-monitor';

const App = () => {
  return (
    <div>
      <h1>Team Activity Monitoring</h1>
      <RealTimeTeamMonitor />
    </div>
  );
};

export default App;
```

This example demonstrates how to integrate the Real-Time Team Activity Monitor into an application, providing an interface for monitoring team operations in real-time.
